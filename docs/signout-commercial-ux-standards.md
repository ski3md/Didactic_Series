# Sign-Out Masterclass UX Standards

These standards apply to breast sign-out and every AP sign-out case generated by the AP contract.

## Product Principles

- The learner must always know where they are: subspecialty, workflow, case title, specimen type, and current reveal state.
- The first meaningful action is visual observation, not answering a quiz.
- A case opens into a single focused workspace. The learner should never search through a long scroll to find the next step.
- The image stage is large, stable, and uniform across cases.
- The final diagnosis and report language are hidden until the learner has completed the observation sequence.
- Resection cases must expose synoptic field practice in the same workspace.
- Gross-only cases must expose gross documentation fields in the same workspace.
- Every control must have a predictable outcome: select case, reveal next step, reset, reveal final diagnosis, open source.
- Learner-facing text must be clinical sign-out language only.

## Journey Budget

- Image visible within 3 clicks from persistent navigation.
- Final report visible within 7 clicks for a standard case.
- The main path must not require hover-only controls, hidden menus, or non-obvious gestures.

## Validation

Run:

```bash
npm run ap:generate-contract
npm run signout:validate-ux
```

The validator walks every AP contract case and every breast sign-out case, checking reveal order, click budget, workflow-specific panels, image evidence requirements, accessibility hooks, and clinical-language requirements.

The report is written to:

```text
src/content/ap_signout/signout_ux_journey_report.json
```
