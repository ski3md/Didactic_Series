# Next 20 Milestones

This plan converts the current roadmap into four implementation tranches. It is meant to be actionable, status-driven, and tied to the current repo state rather than the older single-module framing.

## Tranche 1: Stabilize The Foundations

1. Publish a current-state master roadmap.
2. Replace the stale README with current product and setup guidance.
3. Restore the missing legacy image-manifest compatibility path.
4. Add a deterministic sync script and package command for compatibility assets.
5. Reduce future local runtime churn through improved ignore rules.

## Tranche 2: Consolidate Core Teaching Surfaces

6. Promote core-principles lectures into a first-class didactic module.
7. Split Downloads lectures into topic-first navigation instead of a flat library.
8. Curate one coherent tutorial track from the imported corpora.
9. Deduplicate overlapping AP/CP/tutorial records across primary and Downloads content.
10. Define canonical versus staged content namespaces in code and docs.

## Tranche 3: Build Linking And Navigation

11. Add cross-links between lectures, tutorials, syllabus topics, and atlas entities.
12. Add topic and entity facets to main promoted content views.
13. Add a dedicated algorithm navigator rather than leaving algorithms as raw imports.
14. Normalize image family labels and improve atlas summaries for readability.
15. Add “promote from staging” selection rules so Downloads Library stops being a dumping surface.

## Tranche 4: Harden For Ongoing Use

16. Unify documentation around one setup path and one product definition.
17. Add deterministic validation for normalized content outputs.
18. Separate generated runtime artifacts from canonical repo content.
19. Establish a clean release baseline with reproducible build verification.
20. Publish a release-ready project map describing what is production-like, staged, or experimental.

## Tranche Intent

- Tranche 1 removes structural ambiguity and repo noise.
- Tranche 2 promotes the highest-value educational content.
- Tranche 3 turns imported content into a navigable learning system.
- Tranche 4 makes the project maintainable and releaseable.
